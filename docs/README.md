# `docs/`

Documentation for building and operating the DMRV SaaS platform.

Primary references:
- `../dmrv_saa_s_architecture_near_nft_design.md`
- `../ARCHITECTURE_GAPS_ANALYSIS.md`

Subfolders:
- `architecture/`: C4/system diagrams, architecture extensions, threat models.
- `adr/`: Architecture Decision Records.
- `api/`: OpenAPI/AsyncAPI schemas and API governance.
- `events/`: Event catalog, event schemas, routing patterns, consumer contracts.
- `runbooks/`: Operational runbooks and incident procedures.
- `security/`: Security model, controls, key management, threat modeling.
- `testing/`: Test strategy, environments, data, performance/chaos plans.
- `ci-cd/`: CI/CD design, release strategy, rollback, promotion flow.
- `observability/`: Metrics/logging/tracing standards and dashboards.
- `compliance/`: SOC2/ISO controls mapping, retention, GDPR, audit readiness.
- `data/`: Data retention, archival, lineage, and governance.
- `cost/`: Cost allocation and optimization guidance.
- `integrations/`: Registry + external system integration guides.
